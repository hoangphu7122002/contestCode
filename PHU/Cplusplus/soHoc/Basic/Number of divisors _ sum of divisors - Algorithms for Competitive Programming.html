    Number of divisors / sum of divisors

    In this article we discuss how to compute the number of divisors d(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>d</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$d(n)$ and the sum of divisors (n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi></mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$\sigma(n)$ of a given number n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$.

    Number of divisors

    It should be obvious that the prime factorization of a divisor d<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>d</mi></math>$d$ has to be a subset of the prime factorization of n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$, e.g. 6=23<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>6</mn><mo>=</mo><mn>2</mn><mo></mo><mn>3</mn></math>$6 = 2 \cdot 3$ is a divisor of 60=2235<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>60</mn><mo>=</mo><msup><mn>2</mn><mn>2</mn></msup><mo></mo><mn>3</mn><mo></mo><mn>5</mn></math>$60 = 2^2 \cdot 3 \cdot 5$. So we only need to find all different subsets of the prime factorization of n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$.

    Usually the number of subsets is 2x<math xmlns="http://www.w3.org/1998/Math/MathML"><msup><mn>2</mn><mi>x</mi></msup></math>$2^x$ for a set with x<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>x</mi></math>$x$ elements. However this is no longer true, if there are repeated elements in the set. In our case some prime factors may appear multiple times in the prime factorization of n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$.

    If a prime factor p<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi></math>$p$ appears e<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>e</mi></math>$e$ times in the prime factorization of n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$, then we can use the factor p<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi></math>$p$ up to e<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>e</mi></math>$e$ times in the subset. Which means we have e+1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>e</mi><mo>+</mo><mn>1</mn></math>$e+1$ choices.

    Therefore if the prime factorization of n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$ is p1e1p2e2pkek<math xmlns="http://www.w3.org/1998/Math/MathML"><msubsup><mi>p</mi><mn>1</mn><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mn>1</mn></msub></mrow></msubsup><mo></mo><msubsup><mi>p</mi><mn>2</mn><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mn>2</mn></msub></mrow></msubsup><mo></mo><msubsup><mi>p</mi><mi>k</mi><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mi>k</mi></msub></mrow></msubsup></math>$p_1^{e_1} \cdot p_2^{e_2} \cdots p_k^{e_k}$, where pi<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>p</mi><mi>i</mi></msub></math>$p_i$ are distinct prime numbers, then the number of divisors is:

    d(n)=(e1+1)(e2+1)(ek+1)<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mi>d</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo><mo>=</mo><mo stretchy="false">(</mo><msub><mi>e</mi><mn>1</mn></msub><mo>+</mo><mn>1</mn><mo stretchy="false">)</mo><mo></mo><mo stretchy="false">(</mo><msub><mi>e</mi><mn>2</mn></msub><mo>+</mo><mn>1</mn><mo stretchy="false">)</mo><mo></mo><mo stretchy="false">(</mo><msub><mi>e</mi><mi>k</mi></msub><mo>+</mo><mn>1</mn><mo stretchy="false">)</mo></math>$$d(n) = (e_1 + 1) \cdot (e_2 + 1) \cdots (e_k + 1)$$

    A way of thinking about it is the following:

    • If there is only one distinct prime divisor n=p1e1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi><mo>=</mo><msubsup><mi>p</mi><mn>1</mn><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mn>1</mn></msub></mrow></msubsup></math>$n = p_1^{e_1}$, then there are obviously e1+1<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>e</mi><mn>1</mn></msub><mo>+</mo><mn>1</mn></math>$e_1 + 1$ divisors (1,p1,p12,,p1e1<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>1</mn><mo>,</mo><msub><mi>p</mi><mn>1</mn></msub><mo>,</mo><msubsup><mi>p</mi><mn>1</mn><mn>2</mn></msubsup><mo>,</mo><mo></mo><mo>,</mo><msubsup><mi>p</mi><mn>1</mn><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mn>1</mn></msub></mrow></msubsup></math>$1, p_1, p_1^2, \dots, p_1^{e_1}$).

    • If there are two distinct prime divisors n=p1e1p2e2<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi><mo>=</mo><msubsup><mi>p</mi><mn>1</mn><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mn>1</mn></msub></mrow></msubsup><mo></mo><msubsup><mi>p</mi><mn>2</mn><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mn>2</mn></msub></mrow></msubsup></math>$n = p_1^{e_1} \cdot p_2^{e_2}$, then you can arrange all divisors in form of a tabular.

    • A similar argument can be made if there are more then two distinct prime factors.
    long long numberOfDivisors(long long num) {
    long long total = 1;
    for (int i = 2; (long long)i * i <= num; i++) {
        if (num % i == 0) {
            int e = 0;
            do {
                e++;
                num /= i;
            } while (num % i == 0);
            total *= e + 1;
        }
    }
    if (num > 1) {
        total *= 2;
    }
    return total;
}

    Sum of divisors

    We can use the same argument of the previous section.

    • If there is only one distinct prime divisor n=p1e1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi><mo>=</mo><msubsup><mi>p</mi><mn>1</mn><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mn>1</mn></msub></mrow></msubsup></math>$n = p_1^{e_1}$, then the sum is:
    1+p1+p12++p1e1=p1e1+11p11<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mn>1</mn><mo>+</mo><msub><mi>p</mi><mn>1</mn></msub><mo>+</mo><msubsup><mi>p</mi><mn>1</mn><mn>2</mn></msubsup><mo>+</mo><mo></mo><mo>+</mo><msubsup><mi>p</mi><mn>1</mn><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mn>1</mn></msub></mrow></msubsup><mo>=</mo><mfrac><mrow><msubsup><mi>p</mi><mn>1</mn><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mn>1</mn></msub><mo>+</mo><mn>1</mn></mrow></msubsup><mo></mo><mn>1</mn></mrow><mrow><msub><mi>p</mi><mn>1</mn></msub><mo></mo><mn>1</mn></mrow></mfrac></math>$$1 + p_1 + p_1^2 + \dots + p_1^{e_1} = \frac{p_1^{e_1 + 1} - 1}{p_1 - 1}$$

    long long SumOfDivisors(long long num) {
    long long total = 1;

    for (int i = 2; (long long)i * i <= num; i++) {
        if (num % i == 0) {
            int e = 0;
            do {
                e++;
                num /= i;
            } while (num % i == 0);

            long long sum = 0, pow = 1;
            do {
                sum += pow;
                pow *= i;
            } while (e-- > 0);
            total *= sum;
        }
    }
    if (num > 1) {
        total *= (1 + num);
    }
    return total;
}

    Multiplicative functions

    A multiplicative function is a function f(x)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo></math>$f(x)$ which satisfies

    f(ab)=f(a)f(b)<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mi>f</mi><mo stretchy="false">(</mo><mi>a</mi><mo></mo><mi>b</mi><mo stretchy="false">)</mo><mo>=</mo><mi>f</mi><mo stretchy="false">(</mo><mi>a</mi><mo stretchy="false">)</mo><mo></mo><mi>f</mi><mo stretchy="false">(</mo><mi>b</mi><mo stretchy="false">)</mo></math>$$f(a \cdot b) = f(a) \cdot f(b)$$

    if a<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>a</mi></math>$a$ and b<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>b</mi></math>$b$ are coprime.

    Both d(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>d</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$d(n)$ and (n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi></mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$\sigma(n)$ are multiplicative functions.

    Multiplicative functions have a huge variety of interesting properties, which can be very useful in number theory problems. For instance the Dirichlet convolution of two multiplicative functions is also multiplicative.
