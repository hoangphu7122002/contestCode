    Suffix Array¶

    Definition¶

    Let   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$  be a string of length   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$ . The   i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ -th suffix of   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$  is the substring   s[i…n−1]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi><mo stretchy="false">[</mo><mi>i</mi><mo>…</mo><mi>n</mi><mo>−</mo><mn>1</mn><mo stretchy="false">]</mo></math>$s[i \ldots n - 1]$ .

    A suffix array will contain integers that represent the starting indexes of the all the suffixes of a given string, after the aforementioned suffixes are sorted.

    As an example look at the string   s=abaab<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi><mo>=</mo><mi>a</mi><mi>b</mi><mi>a</mi><mi>a</mi><mi>b</mi></math>$s = abaab$ . All suffixes are as follows

    After sorting these strings:

      2.aab3.ab0.abaab4.b1.baab<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mtable columnalign="left left" columnspacing="1em" rowspacing="4pt"><mtr><mtd><mn>2.</mn></mtd><mtd><mi>a</mi><mi>a</mi><mi>b</mi></mtd></mtr><mtr><mtd><mn>3.</mn></mtd><mtd><mi>a</mi><mi>b</mi></mtd></mtr><mtr><mtd><mn>0.</mn></mtd><mtd><mi>a</mi><mi>b</mi><mi>a</mi><mi>a</mi><mi>b</mi></mtd></mtr><mtr><mtd><mn>4.</mn></mtd><mtd><mi>b</mi></mtd></mtr><mtr><mtd><mn>1.</mn></mtd><mtd><mi>b</mi><mi>a</mi><mi>a</mi><mi>b</mi></mtd></mtr></mtable></math>$$\begin{array}{ll} 2. & aab \\ 3. & ab \\ 0. & abaab \\ 4. & b \\ 1. & baab \end{array}$$ 

    Therefore the suffix array for   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$  will be   (2, 3, 0, 4, 1)<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">(</mo><mn>2</mn><mo>,</mo><mtext>&nbsp;</mtext><mn>3</mn><mo>,</mo><mtext>&nbsp;</mtext><mn>0</mn><mo>,</mo><mtext>&nbsp;</mtext><mn>4</mn><mo>,</mo><mtext>&nbsp;</mtext><mn>1</mn><mo stretchy="false">)</mo></math>$(2,~ 3,~ 0,~ 4,~ 1)$ .

    As a data structure it is widely used in areas such as data compression, bioinformatics and, in general, in any area that deals with strings and string matching problems.

    Construction¶

      O(n2log⁡n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><msup><mi>n</mi><mn>2</mn></msup><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n^2 \log n)$  approach¶

    This is the most naive approach. Get all the suffixes and sort them using quicksort or mergesort and simultaneously retain their original indices. Sorting uses   O(nlog⁡n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n \log n)$  comparisons, and since comparing two strings will additionally take   O(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n)$  time, we get the final complexity of   O(n2log⁡n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><msup><mi>n</mi><mn>2</mn></msup><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n^2 \log n)$ .

      O(nlog⁡n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n \log n)$  approach¶

    Strictly speaking the following algorithm will not sort the suffixes, but rather the cyclic shifts of a string. However we can very easily derive an algorithm for sorting suffixes from it: it is enough to append an arbitrary character to the end of the string which is smaller than any character from the string. It is common to use the symbol $. Then the order of the sorted cyclic shifts is equivalent to the order of the sorted suffixes, as demonstrated here with the string   dabbb<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>d</mi><mi>a</mi><mi>b</mi><mi>b</mi><mi>b</mi></math>$dabbb$ .

    Since we are going to sort cyclic shifts, we will consider cyclic substrings. We will use the notation   s[i…j]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi><mo stretchy="false">[</mo><mi>i</mi><mo>…</mo><mi>j</mi><mo stretchy="false">]</mo></math>$s[i \dots j]$  for the substring of   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$  even if   i>j<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo>&gt;</mo><mi>j</mi></math>$i > j$ . In addition we will take all indices modulo the length of   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$ , and will omit the modulo operation for simplicity.

For example in the   0<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>0</mn></math>$0$ -th iteration the array could also be   p=(3, 1, 0, 2)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo>=</mo><mo stretchy="false">(</mo><mn>3</mn><mo>,</mo><mtext>&nbsp;</mtext><mn>1</mn><mo>,</mo><mtext>&nbsp;</mtext><mn>0</mn><mo>,</mo><mtext>&nbsp;</mtext><mn>2</mn><mo stretchy="false">)</mo></math>$p = (3,~ 1,~ 0,~ 2)$  or   p=(3, 0, 1, 2)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo>=</mo><mo stretchy="false">(</mo><mn>3</mn><mo>,</mo><mtext>&nbsp;</mtext><mn>0</mn><mo>,</mo><mtext>&nbsp;</mtext><mn>1</mn><mo>,</mo><mtext>&nbsp;</mtext><mn>2</mn><mo stretchy="false">)</mo></math>$p = (3,~ 0,~ 1,~ 2)$ . This can be done trivially, for example, by using counting sort. For each character we count how many times it appears in the string, and then use this information to create the array   p[]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$p[]$ .     This gives us a very simple solution: sort the substrings of length   2k<math xmlns="http://www.w3.org/1998/Math/MathML"><msup><mn>2</mn><mi>k</mi></msup></math>$2^k$  by these pairs of numbers. This will give us the required order   p[]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$p[]$ . However a normal sort runs in   O(nlog⁡n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n \log n)$  time, with which we are not satisfied. This will only give us an algorithm for constructing a suffix array in   O(nlog2⁡n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><msup><mi>log</mi><mn>2</mn></msup><mo data-mjx-texclass="NONE">⁡</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n \log^2 n)$  times.

    How do we quickly perform such a sorting of the pairs? Since the elements of the pairs do not exceed   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$ , we can use counting sort again. However sorting pairs with counting sort is not the most efficient. To achieve a better hidden constant in the complexity, we will use another trick.

    We use here the technique on which radix sort is based: to sort the pairs we first sort them by the second element, and then by the first element (with a stable sort, i.e. sorting without breaking the relative order of equal elements). However the second elements were already sorted in the previous iteration. Thus, in order to sort the pairs by the second elements, we just need to subtract   2k−1<math xmlns="http://www.w3.org/1998/Math/MathML"><msup><mn>2</mn><mrow data-mjx-texclass="ORD"><mi>k</mi><mo>−</mo><mn>1</mn></mrow></msup></math>$2^{k-1}$  from the indices in   p[]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$p[]$  (e.g. if the smallest substring of length   2k−1<math xmlns="http://www.w3.org/1998/Math/MathML"><msup><mn>2</mn><mrow data-mjx-texclass="ORD"><mi>k</mi><mo>−</mo><mn>1</mn></mrow></msup></math>$2^{k-1}$  starts at position   i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ , then the substring of length   2k<math xmlns="http://www.w3.org/1998/Math/MathML"><msup><mn>2</mn><mi>k</mi></msup></math>$2^k$  with the smallest second half starts at   i−2k−1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo>−</mo><msup><mn>2</mn><mrow data-mjx-texclass="ORD"><mi>k</mi><mo>−</mo><mn>1</mn></mrow></msup></math>$i - 2^{k-1}$ ).

    So only by simple subtractions we can sort the second elements of the pairs in   p[]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$p[]$ . Now we need to perform a stable sort by the first elements. As already mentioned, this can be accomplished with counting sort.

    The only thing left is to compute the equivalence classes   c[]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>c</mi><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$c[]$ , but as before this can be done by simply iterating over the sorted permutation   p[]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$p[]$  and comparing neighboring pairs.

    Here is the remaining implementation. We use temporary arrays   pn[]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mi>n</mi><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$pn[]$  and   cn[]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>c</mi><mi>n</mi><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$cn[]$  to store the permutation by the second elements and the new equivalent class indices.

        vector<int> pn(n), cn(n);
    for (int h = 0; (1 << h) < n; ++h) {
        for (int i = 0; i < n; i++) {
            pn[i] = p[i] - (1 << h);
            if (pn[i] < 0)
                pn[i] += n;
        }
        fill(cnt.begin(), cnt.begin() + classes, 0);
        for (int i = 0; i < n; i++)
            cnt[c[pn[i]]]++;
        for (int i = 1; i < classes; i++)
            cnt[i] += cnt[i-1];
        for (int i = n-1; i >= 0; i--)
            p[--cnt[c[pn[i]]]] = pn[i];
        cn[p[0]] = 0;
        classes = 1;
        for (int i = 1; i < n; i++) {
            pair<int, int> cur = {c[p[i]], c[(p[i] + (1 << h)) % n]};
            pair<int, int> prev = {c[p[i-1]], c[(p[i-1] + (1 << h)) % n]};
            if (cur != prev)
                ++classes;
            cn[p[i]] = classes - 1;
        }
        c.swap(cn);
    }
    return p;
}
    The algorithm requires   O(nlog⁡n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n \log n)$  time and   O(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n)$  memory. For simplicity we used the complete ASCII range as alphabet.

    If it is known that the string only contains a subset of characters, e.g. only lowercase letters, then the implementation can be optimized, but the optimization factor would likely be insignificant, as the size of the alphabet only matters on the first iteration. Every other iteration depends on the number of equivalence classes, which may quickly reach   O(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n)$  even if initially it was a string over the alphabet of size   2<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>2</mn></math>$2$ .

    Also note, that this algorithm only sorts the cycle shifts. As mentioned at the beginning of this section we can generate the sorted order of the suffixes by appending a character that is smaller than all other characters of the string, and sorting this resulting string by cycle shifts, e.g. by sorting the cycle shifts of $s + $$. This will obviously give the suffix array of   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$ , however prepended with   |s|<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">|</mo><mi>s</mi><mo stretchy="false">|</mo></math>$|s|$ .

    vector<int> suffix_array_construction(string s) {
    s += "$";
    vector<int> sorted_shifts = sort_cyclic_shifts(s);
    sorted_shifts.erase(sorted_shifts.begin());
    return sorted_shifts;
}

    Applications¶

    Finding the smallest cyclic shift¶

    The algorithm above sorts all cyclic shifts (without appending a character to the string), and therefore   p[0]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo stretchy="false">[</mo><mn>0</mn><mo stretchy="false">]</mo></math>$p[0]$  gives the position of the smallest cyclic shift.

    Finding a substring in a string¶

    The task is to find a string   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$  inside some text   t<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>t</mi></math>$t$  online - we know the text   t<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>t</mi></math>$t$  beforehand, but not the string   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$ . We can create the suffix array for the text   t<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>t</mi></math>$t$  in   O(|t|log⁡|t|)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>t</mi><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>t</mi><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mo stretchy="false">)</mo></math>$O(|t| \log |t|)$  time. Now we can look for the substring   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$  in the following way. The occurrence of   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$  must be a prefix of some suffix from   t<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>t</mi></math>$t$ . Since we sorted all the suffixes we can perform a binary search for   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$  in   p<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi></math>$p$ . Comparing the current suffix and the substring   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$  within the binary search can be done in   O(|s|)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>s</mi><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mo stretchy="false">)</mo></math>$O(|s|)$  time, therefore the complexity for finding the substring is   O(|s|log⁡|t|)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>s</mi><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>t</mi><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mo stretchy="false">)</mo></math>$O(|s| \log |t|)$ . Also notice that if the substring occurs multiple times in   t<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>t</mi></math>$t$ , then all occurrences will be next to each other in   p<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi></math>$p$ . Therefore the number of occurrences can be found with a second binary search, and all occurrences can be printed easily.

    Comparing two substrings of a string¶

    We want to be able to compare two substrings of the same length of a given string   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$  in   O(1)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mn>1</mn><mo stretchy="false">)</mo></math>$O(1)$  time, i.e. checking if the first substring is smaller than the second one.

    For this we construct the suffix array in   O(|s|log⁡|s|)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>s</mi><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>s</mi><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mo stretchy="false">)</mo></math>$O(|s| \log |s|)$  time and store all the intermediate results of the equivalence classes   c[]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>c</mi><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$c[]$ .

    Using this information we can compare any two substring whose length is equal to a power of two in O(1): for this it is sufficient to compare the equivalence classes of both substrings. Now we want to generalize this method to substrings of arbitrary length.

    Let's compare two substrings of length   l<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>l</mi></math>$l$  with the starting indices   i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$  and   j<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>j</mi></math>$j$ . We find the largest length of a block that is placed inside a substring of this length: the greatest   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$  such that   2k≤l<math xmlns="http://www.w3.org/1998/Math/MathML"><msup><mn>2</mn><mi>k</mi></msup><mo>≤</mo><mi>l</mi></math>$2^k \le l$ . Then comparing the two substrings can be replaced by comparing two overlapping blocks of length   2k<math xmlns="http://www.w3.org/1998/Math/MathML"><msup><mn>2</mn><mi>k</mi></msup></math>$2^k$ : first you need to compare the two blocks starting at   i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$  and   j<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>j</mi></math>$j$ , and if these are equal then compare the two blocks ending in positions   i+l−1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo>+</mo><mi>l</mi><mo>−</mo><mn>1</mn></math>$i + l - 1$  and   j+l−1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>j</mi><mo>+</mo><mi>l</mi><mo>−</mo><mn>1</mn></math>$j + l - 1$ :

      …si…si+l−2k…si+2k−1…si+l−1⏟2k⏞first…sj…sj+l−2k…sj+2k−1…sj+l−1⏟2k⏞second…<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mo>…</mo><mover><mrow data-mjx-texclass="OP"><mover><mrow><msub><mi>s</mi><mi>i</mi></msub><mo>…</mo><munder><mrow data-mjx-texclass="OP"><munder><mrow><msub><mi>s</mi><mrow data-mjx-texclass="ORD"><mi>i</mi><mo>+</mo><mi>l</mi><mo>−</mo><msup><mn>2</mn><mi>k</mi></msup></mrow></msub><mo>…</mo><msub><mi>s</mi><mrow data-mjx-texclass="ORD"><mi>i</mi><mo>+</mo><msup><mn>2</mn><mi>k</mi></msup><mo>−</mo><mn>1</mn></mrow></msub><mo>…</mo><msub><mi>s</mi><mrow data-mjx-texclass="ORD"><mi>i</mi><mo>+</mo><mi>l</mi><mo>−</mo><mn>1</mn></mrow></msub></mrow><mo>⏟</mo></munder></mrow><mrow data-mjx-texclass="ORD"><msup><mn>2</mn><mi>k</mi></msup></mrow></munder></mrow><mo>⏞</mo></mover></mrow><mrow data-mjx-texclass="ORD"><mtext>first</mtext></mrow></mover><mo>…</mo><mover><mrow data-mjx-texclass="OP"><mover><mrow><msub><mi>s</mi><mi>j</mi></msub><mo>…</mo><munder><mrow data-mjx-texclass="OP"><munder><mrow><msub><mi>s</mi><mrow data-mjx-texclass="ORD"><mi>j</mi><mo>+</mo><mi>l</mi><mo>−</mo><msup><mn>2</mn><mi>k</mi></msup></mrow></msub><mo>…</mo><msub><mi>s</mi><mrow data-mjx-texclass="ORD"><mi>j</mi><mo>+</mo><msup><mn>2</mn><mi>k</mi></msup><mo>−</mo><mn>1</mn></mrow></msub><mo>…</mo><msub><mi>s</mi><mrow data-mjx-texclass="ORD"><mi>j</mi><mo>+</mo><mi>l</mi><mo>−</mo><mn>1</mn></mrow></msub></mrow><mo>⏟</mo></munder></mrow><mrow data-mjx-texclass="ORD"><msup><mn>2</mn><mi>k</mi></msup></mrow></munder></mrow><mo>⏞</mo></mover></mrow><mrow data-mjx-texclass="ORD"><mtext>second</mtext></mrow></mover><mo>…</mo></math>$$\dots \overbrace{s_i \dots \underbrace{s_{i+l-2^k} \dots s_{i+2^k-1} \dots s_{i+l-1}}_{2^k}}^{\text{first}} \dots \overbrace{s_j \dots \underbrace{s_{j+l-2^k} \dots s_{j+2^k-1} \dots s_{j+l-1}}_{2^k}}^{\text{second}} \dots$$ 

    Here is the implementation of the comparison. Note that it is assumed that the function gets called with the already calculated   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$ .   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$  can be computed with   ⌊log⁡l⌋<math xmlns="http://www.w3.org/1998/Math/MathML"><mo fence="false" stretchy="false">⌊</mo><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mi>l</mi><mo fence="false" stretchy="false">⌋</mo></math>$\lfloor \log l \rfloor$ , but it is more efficient to precompute all   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$  values for every   l<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>l</mi></math>$l$ . See for instance the article about the Sparse Table, which uses a similar idea and computes all   log<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>log</mi></math>$\log$  values.

    int compare(int i, int j, int l, int k) {
    pair<int, int> a = {c[k][i], c[k][(i+l-(1 << k))%n]};
    pair<int, int> b = {c[k][j], c[k][(j+l-(1 << k))%n]};
    return a == b ? 0 : a < b ? -1 : 1;
}

    Longest common prefix of two substrings with additional memory¶

    For a given string   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$  we want to compute the longest common prefix (LCP) of two arbitrary suffixes with position   i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$  and   j<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>j</mi></math>$j$ .

    The method described here uses   O(|s|log⁡|s|)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>s</mi><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>s</mi><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mo stretchy="false">)</mo></math>$O(|s| \log |s|)$  additional memory. A completely different approach that will only use a linear amount of memory is described in the next section.

    We construct the suffix array in   O(|s|log⁡|s|)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>s</mi><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>s</mi><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mo stretchy="false">)</mo></math>$O(|s| \log |s|)$  time, and remember the intermediate results of the arrays   c[]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>c</mi><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$c[]$  from each iteration.

    Let's compute the LCP for two suffixes starting at   i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$  and   j<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>j</mi></math>$j$ . We can compare any two substrings with a length equal to a power of two in   O(1)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mn>1</mn><mo stretchy="false">)</mo></math>$O(1)$ . To do this, we compare the strings by power of twos (from highest to lowest power) and if the substrings of this length are the same, then we add the equal length to the answer and continue checking for the LCP to the right of the equal part, i.e.   i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$  and   j<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>j</mi></math>$j$  get added by the current power of two.

    int lcp(int i, int j) {
    int ans = 0;
    for (int k = log_n; k >= 0; k--) {
        if (c[k][i % n] == c[k][j % n]) {
            ans += 1 << k;
            i += 1 << k;
            j += 1 << k;
        }
    }
    return ans;
}

    Here log_n denotes a constant that is equal to the logarithm of   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$  in base   2<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>2</mn></math>$2$  rounded down.

    Longest common prefix of two substrings without additional memory¶

    We have the same task as in the previous section. We have compute the longest common prefix (LCP) for two suffixes of a string   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$ .

    Unlike the previous method this one will only use   O(|s|)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mi>s</mi><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mo stretchy="false">)</mo></math>$O(|s|)$  memory. The result of the preprocessing will be an array (which itself is an important source of information about the string, and therefore also used to solve other tasks). LCP queries can be answered by performing RMQ queries (range minimum queries) in this array, so for different implementations it is possible to achieve logarithmic and even constant query time.

    The basis for this algorithm is the following idea: we will compute the longest common prefix for each pair of adjacent suffixes in the sorted order. In other words we construct an array   lcp[0…n−2]<math xmlns="http://www.w3.org/1998/Math/MathML"><mtext>lcp</mtext><mo stretchy="false">[</mo><mn>0</mn><mo>…</mo><mi>n</mi><mo>−</mo><mn>2</mn><mo stretchy="false">]</mo></math>$\text{lcp}[0 \dots n-2]$ , where   lcp[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><mtext>lcp</mtext><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$\text{lcp}[i]$  is equal to the length of the longest common prefix of the suffixes starting at   p[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$p[i]$  and   p[i+1]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo stretchy="false">[</mo><mi>i</mi><mo>+</mo><mn>1</mn><mo stretchy="false">]</mo></math>$p[i+1]$ . This array will give us an answer for any two adjacent suffixes of the string. Then the answer for arbitrary two suffixes, not necessarily neighboring ones, can be obtained from this array. In fact, let the request be to compute the LCP of the suffixes   p[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$p[i]$  and   p[j]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo stretchy="false">[</mo><mi>j</mi><mo stretchy="false">]</mo></math>$p[j]$ . Then the answer to this query will be   min(lcp[i], lcp[i+1], …, lcp[j−1])<math xmlns="http://www.w3.org/1998/Math/MathML"><mo data-mjx-texclass="OP" movablelimits="true">min</mo><mo stretchy="false">(</mo><mi>l</mi><mi>c</mi><mi>p</mi><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo><mo>,</mo><mtext>&nbsp;</mtext><mi>l</mi><mi>c</mi><mi>p</mi><mo stretchy="false">[</mo><mi>i</mi><mo>+</mo><mn>1</mn><mo stretchy="false">]</mo><mo>,</mo><mtext>&nbsp;</mtext><mo>…</mo><mo>,</mo><mtext>&nbsp;</mtext><mi>l</mi><mi>c</mi><mi>p</mi><mo stretchy="false">[</mo><mi>j</mi><mo>−</mo><mn>1</mn><mo stretchy="false">]</mo><mo stretchy="false">)</mo></math>$\min(lcp[i],~ lcp[i+1],~ \dots,~ lcp[j-1])$ .

    Thus if we have such an array   lcp<math xmlns="http://www.w3.org/1998/Math/MathML"><mtext>lcp</mtext></math>$\text{lcp}$ , then the problem is reduced to the RMQ, which has many wide number of different solutions with different complexities.

    So the main task is to build this array   lcp<math xmlns="http://www.w3.org/1998/Math/MathML"><mtext>lcp</mtext></math>$\text{lcp}$ . We will use Kasai's algorithm, which can compute this array in   O(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n)$  time.

    Let's look at two adjacent suffixes in the sorted order (order of the suffix array). Let their starting positions be   i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$  and   j<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>j</mi></math>$j$  and their   lcp<math xmlns="http://www.w3.org/1998/Math/MathML"><mtext>lcp</mtext></math>$\text{lcp}$  equal to   k>0<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi><mo>&gt;</mo><mn>0</mn></math>$k > 0$ . If we remove the first letter of both suffixes - i.e. we take the suffixes   i+1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo>+</mo><mn>1</mn></math>$i+1$  and   j+1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>j</mi><mo>+</mo><mn>1</mn></math>$j+1$  - then it should be obvious that the   lcp<math xmlns="http://www.w3.org/1998/Math/MathML"><mtext>lcp</mtext></math>$\text{lcp}$  of these two is   k−1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi><mo>−</mo><mn>1</mn></math>$k - 1$ . However we cannot use this value and write it in the   lcp<math xmlns="http://www.w3.org/1998/Math/MathML"><mtext>lcp</mtext></math>$\text{lcp}$  array, because these two suffixes might not be next to each other in the sorted order. The suffix   i+1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo>+</mo><mn>1</mn></math>$i+1$  will of course be smaller than the suffix   j+1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>j</mi><mo>+</mo><mn>1</mn></math>$j+1$ , but there might be some suffixes between them. However, since we know that the LCP between two suffixes is the minimum value of all transitions, we also know that the LCP between any two pairs in that interval has to be at least   k−1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi><mo>−</mo><mn>1</mn></math>$k-1$ , especially also between   i+1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo>+</mo><mn>1</mn></math>$i+1$  and the next suffix. And possibly it can be bigger.

    Now we already can implement the algorithm. We will iterate over the suffixes in order of their length. This way we can reuse the last value   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$ , since going from suffix   i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$  to the suffix   i+1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo>+</mo><mn>1</mn></math>$i+1$  is exactly the same as removing the first letter. We will need an additional array   rank<math xmlns="http://www.w3.org/1998/Math/MathML"><mtext>rank</mtext></math>$\text{rank}$ , which will give us the position of a suffix in the sorted list of suffixes.

    vector<int> lcp_construction(string const& s, vector<int> const& p) {
    int n = s.size();
    vector<int> rank(n, 0);
    for (int i = 0; i < n; i++)
        rank[p[i]] = i;

    int k = 0;
    vector<int> lcp(n-1, 0);
    for (int i = 0; i < n; i++) {
        if (rank[i] == n - 1) {
            k = 0;
            continue;
        }
        int j = p[rank[i] + 1];
        while (i + k < n && j + k < n && s[i+k] == s[j+k])
            k++;
        lcp[rank[i]] = k;
        if (k)
            k--;
    }
    return lcp;
}

    It is easy to see, that we decrease   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$  at most   O(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n)$  times (each iteration at most once, except for   rank[i]==n−1<math xmlns="http://www.w3.org/1998/Math/MathML"><mtext>rank</mtext><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo><mo>==</mo><mi>n</mi><mo>−</mo><mn>1</mn></math>$\text{rank}[i] == n-1$ , where we directly reset it to   0<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>0</mn></math>$0$ ), and the LCP between two strings is at most   n−1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi><mo>−</mo><mn>1</mn></math>$n-1$ , we will also increase   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$  only   O(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n)$  times. Therefore the algorithm runs in   O(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n)$  time.

    Number of different substrings¶

    We preprocess the string   s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$  by computing the suffix array and the LCP array. Using this information we can compute the number of different substrings in the string.

    To do this, we will think about which new substrings begin at position   p[0]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo stretchy="false">[</mo><mn>0</mn><mo stretchy="false">]</mo></math>$p[0]$ , then at   p[1]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo stretchy="false">[</mo><mn>1</mn><mo stretchy="false">]</mo></math>$p[1]$ , etc. In fact we take the suffixes in sorted order and see what prefixes give new substrings.
