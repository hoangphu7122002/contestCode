    It is believed that this formula, as well as the triangle which allows efficient calculation of the coefficients, was discovered by Blaise Pascal in the 17th century. Nevertheless, it was known to the Chinese mathematician Yang Hui, who lived in the 13th century. Perhaps it was discovered by a Persian scholar Omar Khayyam. Moreover, Indian mathematician Pingala, who lived earlier in the 3rd. BC, got similar results. The merit of the Newton is that he generalized this formula for exponents that are not natural.

    Calculation¶

    Analytic formula for the calculation:

      (nk)=n!k!(n−k)!<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>=</mo><mfrac><mrow><mi>n</mi><mo>!</mo></mrow><mrow><mi>k</mi><mo>!</mo><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo><mo>!</mo></mrow></mfrac></math>$$ \binom n k = \frac {n!} {k!(n-k)!} $$ 

    This formula can be easily deduced from the problem of ordered arrangement (number of ways to select   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$  different elements from   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$  different elements). First, let's count the number of ordered selections of   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$  elements. There are   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$  ways to select the first element,   n−1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi><mo>−</mo><mn>1</mn></math>$n-1$  ways to select the second element,   n−2<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi><mo>−</mo><mn>2</mn></math>$n-2$  ways to select the third element, and so on. As a result, we get the formula of the number of ordered arrangements:   n(n−1)(n−2)⋯(n−k+1)=n!(n−k)!<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mn>1</mn><mo stretchy="false">)</mo><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mn>2</mn><mo stretchy="false">)</mo><mo>⋯</mo><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo>+</mo><mn>1</mn><mo stretchy="false">)</mo><mo>=</mo><mfrac><mrow><mi>n</mi><mo>!</mo></mrow><mrow><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo><mo>!</mo></mrow></mfrac></math>$n (n-1) (n-2) \cdots (n - k + 1) = \frac {n!} {(n-k)!}$ . We can easily move to unordered arrangements, noting that each unordered arrangement corresponds to exactly   k!<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi><mo>!</mo></math>$k!$  ordered arrangements (  k!<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi><mo>!</mo></math>$k!$  is the number of possible permutations of   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$  elements). We get the final formula by dividing   n!(n−k)!<math xmlns="http://www.w3.org/1998/Math/MathML"><mfrac><mrow><mi>n</mi><mo>!</mo></mrow><mrow><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo><mo>!</mo></mrow></mfrac></math>$\frac {n!} {(n-k)!}$  by   k!<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi><mo>!</mo></math>$k!$ .

    Recurrence formula (which is associated with the famous "Pascal's Triangle"):

      (nk)=(n−1k−1)+(n−1k)<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>=</mo><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mrow><mi>n</mi><mo>−</mo><mn>1</mn></mrow><mrow><mi>k</mi><mo>−</mo><mn>1</mn></mrow></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>+</mo><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mrow><mi>n</mi><mo>−</mo><mn>1</mn></mrow><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow></math>$$ \binom n k = \binom {n-1} {k-1} + \binom {n-1} k $$ 

    It is easy to deduce this using the analytic formula.

    Note that for   n<k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi><mo>&lt;</mo><mi>k</mi></math>$n \lt k$  the value of   (nk)<math xmlns="http://www.w3.org/1998/Math/MathML"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="1.2em" maxsize="1.2em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="1.2em" maxsize="1.2em">)</mo></mrow></mrow></math>$\binom n k$  is assumed to be zero.

    Properties¶

    Binomial coefficients have many different properties. Here are the simplest of them:

    • Symmetry rule:

        (nk)=(nn−k)<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>=</mo><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mrow><mi>n</mi><mo>−</mo><mi>k</mi></mrow></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow></math>$$ \binom n k = \binom n {n-k} $$ 

    • Factoring in:

        (nk)=nk(n−1k−1)<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>=</mo><mfrac><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mrow><mi>n</mi><mo>−</mo><mn>1</mn></mrow><mrow><mi>k</mi><mo>−</mo><mn>1</mn></mrow></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow></math>$$ \binom n k = \frac n k \binom {n-1} {k-1} $$ 

    • Sum over   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$ :

        ∑k=0n(nk)=2n<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><munderover><mo data-mjx-texclass="OP">∑</mo><mrow data-mjx-texclass="ORD"><mi>k</mi><mo>=</mo><mn>0</mn></mrow><mi>n</mi></munderover><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>=</mo><msup><mn>2</mn><mi>n</mi></msup></math>$$ \sum_{k = 0}^n \binom n k = 2 ^ n $$ 

    • Sum over   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$ :

        ∑m=0n(mk)=(n+1k+1)<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><munderover><mo data-mjx-texclass="OP">∑</mo><mrow data-mjx-texclass="ORD"><mi>m</mi><mo>=</mo><mn>0</mn></mrow><mi>n</mi></munderover><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mi>m</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>=</mo><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mrow><mi>n</mi><mo>+</mo><mn>1</mn></mrow><mrow><mi>k</mi><mo>+</mo><mn>1</mn></mrow></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow></math>$$ \sum_{m = 0}^n \binom m k = \binom {n + 1} {k + 1} $$ 

    • Sum over   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$  and   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$ :

        ∑k=0m(n+kk)=(n+m+1m)<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><munderover><mo data-mjx-texclass="OP">∑</mo><mrow data-mjx-texclass="ORD"><mi>k</mi><mo>=</mo><mn>0</mn></mrow><mi>m</mi></munderover><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mrow><mi>n</mi><mo>+</mo><mi>k</mi></mrow><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>=</mo><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mrow><mi>n</mi><mo>+</mo><mi>m</mi><mo>+</mo><mn>1</mn></mrow><mi>m</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow></math>$$ \sum_{k = 0}^m \binom {n + k} k = \binom {n + m + 1} m $$

    • Weighted sum:

    • Connection with the Fibonacci numbers:

        (n0)+(n−11)+⋯+(n−kk)+⋯+(0n)=Fn+1<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mn>0</mn></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>+</mo><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mrow><mi>n</mi><mo>−</mo><mn>1</mn></mrow><mn>1</mn></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>+</mo><mo>⋯</mo><mo>+</mo><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mrow><mi>n</mi><mo>−</mo><mi>k</mi></mrow><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>+</mo><mo>⋯</mo><mo>+</mo><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mn>0</mn><mi>n</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>=</mo><msub><mi>F</mi><mrow data-mjx-texclass="ORD"><mi>n</mi><mo>+</mo><mn>1</mn></mrow></msub></math>$$ \binom n 0 + \binom {n-1} 1 + \cdots + \binom {n-k} k + \cdots + \binom 0 n = F_{n+1} $$ 

    Calculation¶

    Straightforward calculation using analytical formula¶

    The first, straightforward formula is very easy to code, but this method is likely to overflow even for relatively small values of   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$  and   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$  (even if the answer completely fit into some datatype, the calculation of the intermediate factorials can lead to overflow). Therefore, this method often can only be used with long arithmetic:

    int C(int n, int k) {
    int res = 1;
    for (int i = n - k + 1; i <= n; ++i)
        res *= i;
    for (int i = 2; i <= k; ++i)
        res /= i;
    return res;
}

    Improved implementation¶

    Note that in the above implementation numerator and denominator have the same number of factors (  k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$ ), each of which is greater than or equal to 1. Therefore, we can replace our fraction with a product   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$  fractions, each of which is real-valued. However, on each step after multiplying current answer by each of the next fractions the answer will still be integer (this follows from the property of factoring in).

    C++ implementation:

    int C(int n, int k) {
    double res = 1;
    for (int i = 1; i <= k; ++i)
        res = res * (n - k + i) / i;
    return (int)(res + 0.01);
}

    Here we carefully cast the floating point number to an integer, taking into account that due to the accumulated errors, it may be slightly less than the true value (for example,   2.99999<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>2.99999</mn></math>$2.99999$  instead of   3<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>3</mn></math>$3$ ).

    Pascal's Triangle¶

    By using the recurrence relation we can construct a table of binomial coefficients (Pascal's triangle) and take the result from it. The advantage of this method is that intermediate results never exceed the answer and calculating each new table element requires only one addition. The flaw is slow execution for large   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$  and   k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$  if you just need a single value and not the whole table (because in order to calculate   (nk)<math xmlns="http://www.w3.org/1998/Math/MathML"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="1.2em" maxsize="1.2em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="1.2em" maxsize="1.2em">)</mo></mrow></mrow></math>$\binom n k$  you will need to build a table of all   (ij),1≤i≤n,1≤j≤n<math xmlns="http://www.w3.org/1998/Math/MathML"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="1.2em" maxsize="1.2em">(</mo></mrow><mfrac linethickness="0"><mi>i</mi><mi>j</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="1.2em" maxsize="1.2em">)</mo></mrow></mrow><mo>,</mo><mn>1</mn><mo>≤</mo><mi>i</mi><mo>≤</mo><mi>n</mi><mo>,</mo><mn>1</mn><mo>≤</mo><mi>j</mi><mo>≤</mo><mi>n</mi></math>$\binom i j, 1 \le i \le n, 1 \le j \le n$ , or at least to   1≤j≤min(i,2k)<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>1</mn><mo>≤</mo><mi>j</mi><mo>≤</mo><mo data-mjx-texclass="OP" movablelimits="true">min</mo><mo stretchy="false">(</mo><mi>i</mi><mo>,</mo><mn>2</mn><mi>k</mi><mo stretchy="false">)</mo></math>$1 \le j \le \min (i, 2k)$ ). The time complexity can be considered to be   O(n2)<math xmlns="http://www.w3.org/1998/Math/MathML"><mrow data-mjx-texclass="ORD"><mi data-mjx-variant="-tex-calligraphic" mathvariant="script">O</mi></mrow><mo stretchy="false">(</mo><msup><mi>n</mi><mn>2</mn></msup><mo stretchy="false">)</mo></math>$\mathcal{O}(n^2)$ .

    C++ implementation:

    const int maxn = ...;
int C[maxn + 1][maxn + 1];
C[0][0] = 1;
for (int n = 1; n <= maxn; ++n) {
    C[n][0] = C[n][n] = 1;
    for (int k = 1; k < n; ++k)
        C[n][k] = C[n - 1][k - 1] + C[n - 1][k];
}

    If the entire table of values is not necessary, storing only two last rows of it is sufficient (current   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$ -th row and the previous   n−1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi><mo>−</mo><mn>1</mn></math>$n-1$ -th).

    Calculation in   O(1)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mn>1</mn><mo stretchy="false">)</mo></math>$O(1)$ ¶

    Finally, in some situations it is beneficial to precompute all the factorials in order to produce any necessary binomial coefficient with only two divisions later. This can be advantageous when using long arithmetic, when the memory does not allow precomputation of the whole Pascal's triangle.

    Computing binomial coefficients modulo   m<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi></math>$m$ ¶

    Quite often you come across the problem of computing binomial coefficients modulo some   m<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi></math>$m$ .

    Binomial coefficient for small   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$ ¶

    The previously discussed approach of Pascal's triangle can be used to calculate all values of   (nk)modm<math xmlns="http://www.w3.org/1998/Math/MathML"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="1.2em" maxsize="1.2em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="1.2em" maxsize="1.2em">)</mo></mrow></mrow><mo lspace="thickmathspace" rspace="thickmathspace">mod</mo><mi>m</mi></math>$\binom{n}{k} \bmod m$  for reasonably small   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$ , since it requires time complexity   O(n2)<math xmlns="http://www.w3.org/1998/Math/MathML"><mrow data-mjx-texclass="ORD"><mi data-mjx-variant="-tex-calligraphic" mathvariant="script">O</mi></mrow><mo stretchy="false">(</mo><msup><mi>n</mi><mn>2</mn></msup><mo stretchy="false">)</mo></math>$\mathcal{O}(n^2)$ . This approach can handle any modulo, since only addition operations are used.

    Binomial coefficient modulo large prime¶

    The formula for the binomial coefficients is

      (nk)=n!k!(n−k)!,<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>=</mo><mfrac><mrow><mi>n</mi><mo>!</mo></mrow><mrow><mi>k</mi><mo>!</mo><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo><mo>!</mo></mrow></mfrac><mo>,</mo></math>$$\binom n k = \frac {n!} {k!(n-k)!},$$ 

    so if we want to compute it modulo some prime   m>n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi><mo>&gt;</mo><mi>n</mi></math>$m > n$  we get

      (nk)≡n!⋅(k!)−1⋅((n−k)!)−1modm.<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>≡</mo><mi>n</mi><mo>!</mo><mo>⋅</mo><mo stretchy="false">(</mo><mi>k</mi><mo>!</mo><msup><mo stretchy="false">)</mo><mrow data-mjx-texclass="ORD"><mo>−</mo><mn>1</mn></mrow></msup><mo>⋅</mo><mo stretchy="false">(</mo><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo><mo>!</mo><msup><mo stretchy="false">)</mo><mrow data-mjx-texclass="ORD"><mo>−</mo><mn>1</mn></mrow></msup><mspace width="1em"></mspace><mi>mod</mi><mstyle scriptlevel="0"><mspace width="0.167em"></mspace></mstyle><mstyle scriptlevel="0"><mspace width="0.167em"></mspace></mstyle><mi>m</mi><mo>.</mo></math>$$\binom n k \equiv n! \cdot (k!)^{-1} \cdot ((n-k)!)^{-1} \mod m.$$ 

    First we precompute all factorials modulo   m<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi></math>$m$  up to   MAXN!<math xmlns="http://www.w3.org/1998/Math/MathML"><mtext>MAXN</mtext><mo>!</mo></math>$\text{MAXN}!$  in   O(MAXN)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mtext>MAXN</mtext><mo stretchy="false">)</mo></math>$O(\text{MAXN})$  time.

    factorial[0] = 1;
for (int i = 1; i <= MAXN; i++) {
    factorial[i] = factorial[i - 1] * i % m;
}

    And afterwards we can compute the binomial coefficient in   O(log⁡m)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mi>m</mi><mo stretchy="false">)</mo></math>$O(\log m)$  time.

    long long binomial_coefficient(int n, int k) {
    return factorial[n] * inverse(factorial[k] * factorial[n - k] % m) % m;
}

    We even can compute the binomial coefficient in   O(1)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mn>1</mn><mo stretchy="false">)</mo></math>$O(1)$  time if we precompute the inverses of all factorials in   O(MAXNlog⁡m)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mtext>MAXN</mtext><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mi>m</mi><mo stretchy="false">)</mo></math>$O(\text{MAXN} \log m)$  using the regular method for computing the inverse, or even in   O(MAXN)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mtext>MAXN</mtext><mo stretchy="false">)</mo></math>$O(\text{MAXN})$  time using the congruence   (x!)−1≡((x−1)!)−1⋅x−1<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">(</mo><mi>x</mi><mo>!</mo><msup><mo stretchy="false">)</mo><mrow data-mjx-texclass="ORD"><mo>−</mo><mn>1</mn></mrow></msup><mo>≡</mo><mo stretchy="false">(</mo><mo stretchy="false">(</mo><mi>x</mi><mo>−</mo><mn>1</mn><mo stretchy="false">)</mo><mo>!</mo><msup><mo stretchy="false">)</mo><mrow data-mjx-texclass="ORD"><mo>−</mo><mn>1</mn></mrow></msup><mo>⋅</mo><msup><mi>x</mi><mrow data-mjx-texclass="ORD"><mo>−</mo><mn>1</mn></mrow></msup></math>$(x!)^{-1} \equiv ((x-1)!)^{-1} \cdot x^{-1}$  and the method for computing all inverses in   O(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n)$ .

    long long binomial_coefficient(int n, int k) {
    return factorial[n] * inverse_factorial[k] % m * inverse_factorial[n - k] % m;
}

    Binomial coefficient modulo prime power¶

    Here we want to compute the binomial coefficient modulo some prime power, i.e.   m=pb<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi><mo>=</mo><msup><mi>p</mi><mi>b</mi></msup></math>$m = p^b$  for some prime   p<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi></math>$p$ . If   p>max(k,n−k)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo>&gt;</mo><mo data-mjx-texclass="OP" movablelimits="true">max</mo><mo stretchy="false">(</mo><mi>k</mi><mo>,</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo></math>$p > \max(k, n-k)$ , then we can use the same method as described in the previous section. But if   p≤max(k,n−k)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi><mo>≤</mo><mo data-mjx-texclass="OP" movablelimits="true">max</mo><mo stretchy="false">(</mo><mi>k</mi><mo>,</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo></math>$p \le \max(k, n-k)$ , then at least one of   k!<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi><mo>!</mo></math>$k!$  and   (n−k)!<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo><mo>!</mo></math>$(n-k)!$  are not coprime with   m<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi></math>$m$ , and therefore we cannot compute the inverses - they don't exist. Nevertheless we can compute the binomial coefficient.

    The idea is the following: We compute for each   x!<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>x</mi><mo>!</mo></math>$x!$  the biggest exponent   c<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>c</mi></math>$c$  such that   pc<math xmlns="http://www.w3.org/1998/Math/MathML"><msup><mi>p</mi><mi>c</mi></msup></math>$p^c$  divides   x!<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>x</mi><mo>!</mo></math>$x!$ , i.e.   pc | x!<math xmlns="http://www.w3.org/1998/Math/MathML"><msup><mi>p</mi><mi>c</mi></msup><mtext>&nbsp;</mtext><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mtext>&nbsp;</mtext><mi>x</mi><mo>!</mo></math>$p^c ~|~ x!$ . Let   c(x)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>c</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo></math>$c(x)$  be that number. And let   g(x):=x!pc(x)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo><mo>:=</mo><mfrac><mrow><mi>x</mi><mo>!</mo></mrow><msup><mi>p</mi><mrow data-mjx-texclass="ORD"><mi>c</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo></mrow></msup></mfrac></math>$g(x) := \frac{x!}{p^{c(x)}}$ . Then we can write the binomial coefficient as:

      (nk)=g(n)pc(n)g(k)pc(k)g(n−k)pc(n−k)=g(n)g(k)g(n−k)pc(n)−c(k)−c(n−k)<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="2.047em" maxsize="2.047em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="2.047em" maxsize="2.047em">)</mo></mrow></mrow><mo>=</mo><mfrac><mrow><mi>g</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo><msup><mi>p</mi><mrow data-mjx-texclass="ORD"><mi>c</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></mrow></msup></mrow><mrow><mi>g</mi><mo stretchy="false">(</mo><mi>k</mi><mo stretchy="false">)</mo><msup><mi>p</mi><mrow data-mjx-texclass="ORD"><mi>c</mi><mo stretchy="false">(</mo><mi>k</mi><mo stretchy="false">)</mo></mrow></msup><mi>g</mi><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo><msup><mi>p</mi><mrow data-mjx-texclass="ORD"><mi>c</mi><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo></mrow></msup></mrow></mfrac><mo>=</mo><mfrac><mrow><mi>g</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></mrow><mrow><mi>g</mi><mo stretchy="false">(</mo><mi>k</mi><mo stretchy="false">)</mo><mi>g</mi><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo></mrow></mfrac><msup><mi>p</mi><mrow data-mjx-texclass="ORD"><mi>c</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo><mo>−</mo><mi>c</mi><mo stretchy="false">(</mo><mi>k</mi><mo stretchy="false">)</mo><mo>−</mo><mi>c</mi><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo></mrow></msup></math>$$\binom n k = \frac {g(n) p^{c(n)}} {g(k) p^{c(k)} g(n-k) p^{c(n-k)}} = \frac {g(n)} {g(k) g(n-k)}p^{c(n) - c(k) - c(n-k)}$$ 

    The interesting thing is, that   g(x)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo></math>$g(x)$  is now free from the prime divisor   p<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi></math>$p$ . Therefore   g(x)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo></math>$g(x)$  is coprime to m, and we can compute the modular inverses of   g(k)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi><mo stretchy="false">(</mo><mi>k</mi><mo stretchy="false">)</mo></math>$g(k)$  and   g(n−k)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo></math>$g(n-k)$ .

    After precomputing all values for   g<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi></math>$g$  and   c<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>c</mi></math>$c$ , which can be done efficiently using dynamic programming in   O(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mrow data-mjx-texclass="ORD"><mi data-mjx-variant="-tex-calligraphic" mathvariant="script">O</mi></mrow><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$\mathcal{O}(n)$ , we can compute the binomial coefficient in   O(log⁡m)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>log</mi><mo data-mjx-texclass="NONE">⁡</mo><mi>m</mi><mo stretchy="false">)</mo></math>$O(\log m)$  time. Or precompute all inverses and all powers of   p<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>p</mi></math>$p$ , and then compute the binomial coefficient in   O(1)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mn>1</mn><mo stretchy="false">)</mo></math>$O(1)$ .

    Notice, if   c(n)−c(k)−c(n−k)≥b<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>c</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo><mo>−</mo><mi>c</mi><mo stretchy="false">(</mo><mi>k</mi><mo stretchy="false">)</mo><mo>−</mo><mi>c</mi><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo><mo>≥</mo><mi>b</mi></math>$c(n) - c(k) - c(n-k) \ge b$ , than   pb | pc(n)−c(k)−c(n−k)<math xmlns="http://www.w3.org/1998/Math/MathML"><msup><mi>p</mi><mi>b</mi></msup><mtext>&nbsp;</mtext><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mtext>&nbsp;</mtext><msup><mi>p</mi><mrow data-mjx-texclass="ORD"><mi>c</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo><mo>−</mo><mi>c</mi><mo stretchy="false">(</mo><mi>k</mi><mo stretchy="false">)</mo><mo>−</mo><mi>c</mi><mo stretchy="false">(</mo><mi>n</mi><mo>−</mo><mi>k</mi><mo stretchy="false">)</mo></mrow></msup></math>$p^b ~|~ p^{c(n) - c(k) - c(n-k)}$ , and the binomial coefficient is   0<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>0</mn></math>$0$ .

    Binomial coefficient modulo an arbitrary number¶

    Now we compute the binomial coefficient modulo some arbitrary modulus   m<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi></math>$m$ .

    Let the prime factorization of   m<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi></math>$m$  be   m=p1e1p2e2⋯pheh<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi><mo>=</mo><msubsup><mi>p</mi><mn>1</mn><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mn>1</mn></msub></mrow></msubsup><msubsup><mi>p</mi><mn>2</mn><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mn>2</mn></msub></mrow></msubsup><mo>⋯</mo><msubsup><mi>p</mi><mi>h</mi><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mi>h</mi></msub></mrow></msubsup></math>$m = p_1^{e_1} p_2^{e_2} \cdots p_h^{e_h}$ . We can compute the binomial coefficient modulo   piei<math xmlns="http://www.w3.org/1998/Math/MathML"><msubsup><mi>p</mi><mi>i</mi><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mi>i</mi></msub></mrow></msubsup></math>$p_i^{e_i}$  for every   i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ . This gives us   h<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>h</mi></math>$h$  different congruences. Since all moduli   piei<math xmlns="http://www.w3.org/1998/Math/MathML"><msubsup><mi>p</mi><mi>i</mi><mrow data-mjx-texclass="ORD"><msub><mi>e</mi><mi>i</mi></msub></mrow></msubsup></math>$p_i^{e_i}$  are coprime, we can apply the Chinese Remainder Theorem to compute the binomial coefficient modulo the product of the moduli, which is the desired binomial coefficient modulo   m<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi></math>$m$ .

    Binomial coefficient for large   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$  and small modulo¶

    When   n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$  is too large, the   O(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mrow data-mjx-texclass="ORD"><mi data-mjx-variant="-tex-calligraphic" mathvariant="script">O</mi></mrow><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$\mathcal{O}(n)$  algorithms discussed above become impractical. However, if the modulo   m<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi></math>$m$  is small there are still ways to calculate   (nk)modm<math xmlns="http://www.w3.org/1998/Math/MathML"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="1.2em" maxsize="1.2em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="1.2em" maxsize="1.2em">)</mo></mrow></mrow><mo lspace="thickmathspace" rspace="thickmathspace">mod</mo><mi>m</mi></math>$\binom{n}{k} \bmod m$ .

    When the modulo   m<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi></math>$m$  is prime, there are 2 options:

    • Lucas's theorem can be applied which breaks the problem of computing   (nk)modm<math xmlns="http://www.w3.org/1998/Math/MathML"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="1.2em" maxsize="1.2em">(</mo></mrow><mfrac linethickness="0"><mi>n</mi><mi>k</mi></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="1.2em" maxsize="1.2em">)</mo></mrow></mrow><mo lspace="thickmathspace" rspace="thickmathspace">mod</mo><mi>m</mi></math>$\binom{n}{k} \bmod m$  into   logm⁡n<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>log</mi><mi>m</mi></msub><mo data-mjx-texclass="NONE">⁡</mo><mi>n</mi></math>$\log_m n$  problems of the form   (xiyi)modm<math xmlns="http://www.w3.org/1998/Math/MathML"><mrow data-mjx-texclass="ORD"><mrow data-mjx-texclass="OPEN"><mo minsize="1.2em" maxsize="1.2em">(</mo></mrow><mfrac linethickness="0"><msub><mi>x</mi><mi>i</mi></msub><msub><mi>y</mi><mi>i</mi></msub></mfrac><mrow data-mjx-texclass="CLOSE"><mo minsize="1.2em" maxsize="1.2em">)</mo></mrow></mrow><mo lspace="thickmathspace" rspace="thickmathspace">mod</mo><mi>m</mi></math>$\binom{x_i}{y_i} \bmod m$  where   xi,yi<m<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>x</mi><mi>i</mi></msub><mo>,</mo><msub><mi>y</mi><mi>i</mi></msub><mo>&lt;</mo><mi>m</mi></math>$x_i, y_i < m$ . If each reduced coefficient is calculated using precomputed factorials and inverse factorials, the complexity is   O(m+logm⁡n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mrow data-mjx-texclass="ORD"><mi data-mjx-variant="-tex-calligraphic" mathvariant="script">O</mi></mrow><mo stretchy="false">(</mo><mi>m</mi><mo>+</mo><msub><mi>log</mi><mi>m</mi></msub><mo data-mjx-texclass="NONE">⁡</mo><mi>n</mi><mo stretchy="false">)</mo></math>$\mathcal{O}(m + \log_m n)$ .
    • The method of computing factorial modulo P can be used to get the required   g<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi></math>$g$  and   c<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>c</mi></math>$c$  values and use them as described in the section of modulo prime power. This takes   O(mlogm⁡n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mrow data-mjx-texclass="ORD"><mi data-mjx-variant="-tex-calligraphic" mathvariant="script">O</mi></mrow><mo stretchy="false">(</mo><mi>m</mi><msub><mi>log</mi><mi>m</mi></msub><mo data-mjx-texclass="NONE">⁡</mo><mi>n</mi><mo stretchy="false">)</mo></math>$\mathcal{O}(m \log_m n)$ .

    When   m<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi></math>$m$  is not prime but square-free, the prime factors of   m<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi></math>$m$  can be obtained and the coefficient modulo each prime factor can be calculated using either of the above methods, and the overall answer can be obtained by the Chinese Remainder Theorem.

    When   m<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi></math>$m$  is not square-free, a generalization of Lucas's theorem for prime powers can be applied instead of Lucas's theorem.
