    Fenwick Tree

    Let f<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>f</mi></math>$f$ be some group operation (a binary associative function over a set with an identity element and inverse elements) and A<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>A</mi></math>$A$ be an array of integers of length N<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>N</mi></math>$N$. Denote f<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>f</mi></math>$f$'s infix notation as <math xmlns="http://www.w3.org/1998/Math/MathML"><mo></mo></math>$*$; that is, f(x,y)=xy<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo>,</mo><mi>y</mi><mo stretchy="false">)</mo><mo>=</mo><mi>x</mi><mo></mo><mi>y</mi></math>$f(x,y) = x*y$ for arbitrary integers x,y<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>x</mi><mo>,</mo><mi>y</mi></math>$x,y$. (Since this is associative, we will omit parentheses for order of application of f<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>f</mi></math>$f$ when using infix notation.)

    The Fenwick tree is a data structure which:

    • calculates the value of function f<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>f</mi></math>$f$ in the given range [l,r]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mi>l</mi><mo>,</mo><mi>r</mi><mo stretchy="false">]</mo></math>$[l, r]$ (i.e. AlAl+1Ar<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>A</mi><mi>l</mi></msub><mo></mo><msub><mi>A</mi><mrow data-mjx-texclass="ORD"><mi>l</mi><mo>+</mo><mn>1</mn></mrow></msub><mo></mo><mo></mo><mo></mo><msub><mi>A</mi><mi>r</mi></msub></math>$A_l * A_{l+1} * \dots * A_r$) in O(logN)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>log</mi><mo data-mjx-texclass="NONE"></mo><mi>N</mi><mo stretchy="false">)</mo></math>$O(\log N)$ time
    • updates the value of an element of A<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>A</mi></math>$A$ in O(logN)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>log</mi><mo data-mjx-texclass="NONE"></mo><mi>N</mi><mo stretchy="false">)</mo></math>$O(\log N)$ time
    • requires O(N)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>N</mi><mo stretchy="false">)</mo></math>$O(N)$ memory (the same amount required for A<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>A</mi></math>$A$)
    • is easy to use and code, especially in the case of multidimensional arrays

    The most common application of a Fenwick tree is calculating the sum of a range. For example, using addition over the set of integers as the group operation, i.e. f(x,y)=x+y<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo>,</mo><mi>y</mi><mo stretchy="false">)</mo><mo>=</mo><mi>x</mi><mo>+</mo><mi>y</mi></math>$f(x,y) = x + y$: the binary operation, <math xmlns="http://www.w3.org/1998/Math/MathML"><mo></mo></math>$*$, is +<math xmlns="http://www.w3.org/1998/Math/MathML"><mo>+</mo></math>$+$ in this case, so AlAl+1Ar=Al+Al+1++Ar<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>A</mi><mi>l</mi></msub><mo></mo><msub><mi>A</mi><mrow data-mjx-texclass="ORD"><mi>l</mi><mo>+</mo><mn>1</mn></mrow></msub><mo></mo><mo></mo><mo></mo><msub><mi>A</mi><mi>r</mi></msub><mo>=</mo><msub><mi>A</mi><mi>l</mi></msub><mo>+</mo><msub><mi>A</mi><mrow data-mjx-texclass="ORD"><mi>l</mi><mo>+</mo><mn>1</mn></mrow></msub><mo>+</mo><mo></mo><mo>+</mo><msub><mi>A</mi><mrow data-mjx-texclass="ORD"><mi>r</mi></mrow></msub></math>$A_l * A_{l+1} * \dots * A_r = A_l + A_{l+1} + \dots + A_{r}$.

    The Fenwick tree is also called a Binary Indexed Tree (BIT). It was first described in a paper titled "A new data structure for cumulative frequency tables" (Peter M. Fenwick, 1994).

    Description

    Overview

    For the sake of simplicity, we will assume that function f<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>f</mi></math>$f$ is defined as f(x,y)=x+y<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo>,</mo><mi>y</mi><mo stretchy="false">)</mo><mo>=</mo><mi>x</mi><mo>+</mo><mi>y</mi></math>$f(x,y) = x + y$ over the integers.

    Suppose we are given an array of integers, A[0N1]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>A</mi><mo stretchy="false">[</mo><mn>0</mn><mo></mo><mi>N</mi><mo></mo><mn>1</mn><mo stretchy="false">]</mo></math>$A[0 \dots N-1]$. (Note that we are using zero-based indexing.) A Fenwick tree is just an array, T[0N1]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>T</mi><mo stretchy="false">[</mo><mn>0</mn><mo></mo><mi>N</mi><mo></mo><mn>1</mn><mo stretchy="false">]</mo></math>$T[0 \dots N-1]$, where each element is equal to the sum of elements of A<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>A</mi></math>$A$ in some range, [g(i),i]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mi>g</mi><mo stretchy="false">(</mo><mi>i</mi><mo stretchy="false">)</mo><mo>,</mo><mi>i</mi><mo stretchy="false">]</mo></math>$[g(i), i]$:

    Ti=j=g(i)iAj<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><msub><mi>T</mi><mi>i</mi></msub><mo>=</mo><munderover><mo data-mjx-texclass="OP"></mo><mrow data-mjx-texclass="ORD"><mi>j</mi><mo>=</mo><mi>g</mi><mo stretchy="false">(</mo><mi>i</mi><mo stretchy="false">)</mo></mrow><mrow data-mjx-texclass="ORD"><mi>i</mi></mrow></munderover><mrow data-mjx-texclass="ORD"><msub><mi>A</mi><mi>j</mi></msub></mrow></math>$$T_i = \sum_{j = g(i)}^{i}{A_j}$$

    where g<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi></math>$g$ is some function that satisfies 0g(i)i<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>0</mn><mo></mo><mi>g</mi><mo stretchy="false">(</mo><mi>i</mi><mo stretchy="false">)</mo><mo></mo><mi>i</mi></math>$0 \le g(i) \le i$. We will define g<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi></math>$g$ in the next few paragraphs.

    The data structure is called a tree because there is a nice representation of it in the form of a tree, although we don't need to model an actual tree with nodes and edges. We only need to maintain the array T<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>T</mi></math>$T$ to handle all queries.

    Note: The Fenwick tree presented here uses zero-based indexing. Many people use a version of the Fenwick tree that uses one-based indexing. As such, you will also find an alternative implementation which uses one-based indexing in the implementation section. Both versions are equivalent in terms of time and memory complexity.

    Now we can write some pseudo-code for the two operations mentioned above. Below, we get the sum of elements of A<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>A</mi></math>$A$ in the range [0,r]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mn>0</mn><mo>,</mo><mi>r</mi><mo stretchy="false">]</mo></math>$[0, r]$ and update (increase) some element Ai<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>A</mi><mi>i</mi></msub></math>$A_i$:

    def sum(int r):
    res = 0
    while (r >= 0):
        res += t[r]
        r = g(r) - 1
    return res

def increase(int i, int delta):
    for all j with g(j) <= i <= j:
        t[j] += delta

    The function sum works as follows:

    1. First, it adds the sum of the range [g(r),r]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mi>g</mi><mo stretchy="false">(</mo><mi>r</mi><mo stretchy="false">)</mo><mo>,</mo><mi>r</mi><mo stretchy="false">]</mo></math>$[g(r), r]$ (i.e. T[r]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>T</mi><mo stretchy="false">[</mo><mi>r</mi><mo stretchy="false">]</mo></math>$T[r]$) to the result.
    2. Then, it "jumps" to the range [g(g(r)1),g(r)1]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mi>g</mi><mo stretchy="false">(</mo><mi>g</mi><mo stretchy="false">(</mo><mi>r</mi><mo stretchy="false">)</mo><mo></mo><mn>1</mn><mo stretchy="false">)</mo><mo>,</mo><mi>g</mi><mo stretchy="false">(</mo><mi>r</mi><mo stretchy="false">)</mo><mo></mo><mn>1</mn><mo stretchy="false">]</mo></math>$[g(g(r)-1), g(r)-1]$ and adds this range's sum to the result.
    3. This continues until it "jumps" from [0,g(g(g(r)11)1)]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mn>0</mn><mo>,</mo><mi>g</mi><mo stretchy="false">(</mo><mi>g</mi><mo stretchy="false">(</mo><mo></mo><mi>g</mi><mo stretchy="false">(</mo><mi>r</mi><mo stretchy="false">)</mo><mo></mo><mn>1</mn><mo></mo><mo></mo><mn>1</mn><mo stretchy="false">)</mo><mo></mo><mn>1</mn><mo stretchy="false">)</mo><mo stretchy="false">]</mo></math>$[0, g(g( \dots g(r)-1 \dots -1)-1)]$ to [g(1),1]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mi>g</mi><mo stretchy="false">(</mo><mo></mo><mn>1</mn><mo stretchy="false">)</mo><mo>,</mo><mo></mo><mn>1</mn><mo stretchy="false">]</mo></math>$[g(-1), -1]$; this is where the sum function stops jumping.

    The function increase works with the same analogy, but it "jumps" in the direction of increasing indices:

    1. The sum for each range of the form [g(j),j]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mi>g</mi><mo stretchy="false">(</mo><mi>j</mi><mo stretchy="false">)</mo><mo>,</mo><mi>j</mi><mo stretchy="false">]</mo></math>$[g(j), j]$ which satisfies the condition g(j)ij<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi><mo stretchy="false">(</mo><mi>j</mi><mo stretchy="false">)</mo><mo></mo><mi>i</mi><mo></mo><mi>j</mi></math>$g(j) \le i \le j$ is increased by delta; that is, t[j] += delta. Therefore, it updates all elements in T<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>T</mi></math>$T$ that correspond to ranges in which Ai<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>A</mi><mi>i</mi></msub></math>$A_i$ lies.

    The complexity of both sum and increase depend on the function g<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi></math>$g$. There are many ways to choose the function g<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi></math>$g$ such that 0g(i)i<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>0</mn><mo></mo><mi>g</mi><mo stretchy="false">(</mo><mi>i</mi><mo stretchy="false">)</mo><mo></mo><mi>i</mi></math>$0 \le g(i) \le i$ for all i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$. For instance, the function g(i)=i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi><mo stretchy="false">(</mo><mi>i</mi><mo stretchy="false">)</mo><mo>=</mo><mi>i</mi></math>$g(i) = i$ works, which yields T=A<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>T</mi><mo>=</mo><mi>A</mi></math>$T = A$ (in which case, the summation queries are slow). We could also take the function g(i)=0<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi><mo stretchy="false">(</mo><mi>i</mi><mo stretchy="false">)</mo><mo>=</mo><mn>0</mn></math>$g(i) = 0$. This would correspond to prefix sum arrays (in which case, finding the sum of the range [0,i]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mn>0</mn><mo>,</mo><mi>i</mi><mo stretchy="false">]</mo></math>$[0, i]$ will only take constant time; however, updates are slow). The clever part of the algorithm for Fenwick trees is how it uses a special definition of the function g<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi></math>$g$ which can handle both operations in O(logN)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>log</mi><mo data-mjx-texclass="NONE"></mo><mi>N</mi><mo stretchy="false">)</mo></math>$O(\log N)$ time.

    Definition of g(i)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi><mo stretchy="false">(</mo><mi>i</mi><mo stretchy="false">)</mo></math>$g(i)$

    The computation of g(i)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi><mo stretchy="false">(</mo><mi>i</mi><mo stretchy="false">)</mo></math>$g(i)$ is defined using the following simple operation: we replace all trailing 1<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>1</mn></math>$1$ bits in the binary representation of i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ with 0<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>0</mn></math>$0$ bits.

    In other words, if the least significant digit of i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ in binary is 0<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>0</mn></math>$0$, then g(i)=i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi><mo stretchy="false">(</mo><mi>i</mi><mo stretchy="false">)</mo><mo>=</mo><mi>i</mi></math>$g(i) = i$.     Implementation

    Finding sum in one-dimensional array

    Here we present an implementation of the Fenwick tree for sum queries and single updates.

    The normal Fenwick tree can only answer sum queries of the type [0,r]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mn>0</mn><mo>,</mo><mi>r</mi><mo stretchy="false">]</mo></math>$[0, r]$ using sum(int r), however we can also answer other queries of the type [l,r]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mi>l</mi><mo>,</mo><mi>r</mi><mo stretchy="false">]</mo></math>$[l, r]$ by computing two sums [0,r]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mn>0</mn><mo>,</mo><mi>r</mi><mo stretchy="false">]</mo></math>$[0, r]$ and [0,l1]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mn>0</mn><mo>,</mo><mi>l</mi><mo></mo><mn>1</mn><mo stretchy="false">]</mo></math>$[0, l-1]$ and subtract them. This is handled in the sum(int l, int r) method.

    Also this implementation supports two constructors. You can create a Fenwick tree initialized with zeros, or you can convert an existing array into the Fenwick form.

    struct FenwickTree {
    vector<int> bit;  // binary indexed tree
    int n;

    FenwickTree(int n) {
        this->n = n;
        bit.assign(n, 0);
    }

    FenwickTree(vector<int> const &a) : FenwickTree(a.size()) {
        for (size_t i = 0; i < a.size(); i++)
            add(i, a[i]);
    }

    int sum(int r) {
        int ret = 0;
        for (; r >= 0; r = (r & (r + 1)) - 1)
            ret += bit[r];
        return ret;
    }

    int sum(int l, int r) {
        return sum(r) - sum(l - 1);
    }

    void add(int idx, int delta) {
        for (; idx < n; idx = idx | (idx + 1))
            bit[idx] += delta;
    }
};

    Linear construction

    The above implementation requires O(NlogN)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>N</mi><mi>log</mi><mo data-mjx-texclass="NONE"></mo><mi>N</mi><mo stretchy="false">)</mo></math>$O(N \log N)$ time. It's possible to improve that to O(N)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>N</mi><mo stretchy="false">)</mo></math>$O(N)$ time.

    The idea is, that the number a[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>a</mi><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$a[i]$ at index i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ will contribute to the range stored in bit[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>b</mi><mi>i</mi><mi>t</mi><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$bit[i]$, and to all ranges that the index i|(i+1)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mrow data-mjx-texclass="ORD"><mo stretchy="false">|</mo></mrow><mo stretchy="false">(</mo><mi>i</mi><mo>+</mo><mn>1</mn><mo stretchy="false">)</mo></math>$i | (i + 1)$ contributes to. So by adding the numbers in order, you only have to push the current sum further to the next range, where it will then get pushed further to the next range, and so on.

    FenwickTree(vector<int> const &a) : FenwickTree(a.size()){
    for (int i = 0; i < n; i++) {
        bit[i] += a[i];
        int r = i | (i + 1);
        if (r < n) bit[r] += bit[i];
    }
}

    Finding minimum of [0,r]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mn>0</mn><mo>,</mo><mi>r</mi><mo stretchy="false">]</mo></math>$[0, r]$ in one-dimensional array

    It is obvious that there is no easy way of finding minimum of range [l,r]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mi>l</mi><mo>,</mo><mi>r</mi><mo stretchy="false">]</mo></math>$[l, r]$ using Fenwick tree, as Fenwick tree can only answer queries of type [0,r]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mn>0</mn><mo>,</mo><mi>r</mi><mo stretchy="false">]</mo></math>$[0, r]$. Additionally, each time a value is update'd, the new value has to be smaller than the current value. Both significant limitations are because the min<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>m</mi><mi>i</mi><mi>n</mi></math>$min$ operation together with the set of integers doesn't form a group, as there are no inverse elements.

    struct FenwickTreeMin {
    vector<int> bit;
    int n;
    const int INF = (int)1e9;

    FenwickTreeMin(int n) {
        this->n = n;
        bit.assign(n, INF);
    }

    FenwickTreeMin(vector<int> a) : FenwickTreeMin(a.size()) {
        for (size_t i = 0; i < a.size(); i++)
            update(i, a[i]);
    }

    int getmin(int r) {
        int ret = INF;
        for (; r >= 0; r = (r & (r + 1)) - 1)
            ret = min(ret, bit[r]);
        return ret;
    }

    void update(int idx, int val) {
        for (; idx < n; idx = idx | (idx + 1))
            bit[idx] = min(bit[idx], val);
    }
};

    Note: it is possible to implement a Fenwick tree that can handle arbitrary minimum range queries and arbitrary updates. The paper Efficient Range Minimum Queries using Binary Indexed Trees describes such an approach. However with that approach you need to maintain a second binary indexed tree over the data, with a slightly different structure, since one tree is not enough to store the values of all elements in the array. The implementation is also a lot harder compared to the normal implementation for sums.

    Finding sum in two-dimensional array

    As claimed before, it is very easy to implement Fenwick Tree for multidimensional array.

    struct FenwickTree2D {
    vector<vector<int>> bit;
    int n, m;

    // init(...) { ... }

    int sum(int x, int y) {
        int ret = 0;
        for (int i = x; i >= 0; i = (i & (i + 1)) - 1)
            for (int j = y; j >= 0; j = (j & (j + 1)) - 1)
                ret += bit[i][j];
        return ret;
    }

    void add(int x, int y, int delta) {
        for (int i = x; i < n; i = i | (i + 1))
            for (int j = y; j < m; j = j | (j + 1))
                bit[i][j] += delta;
    }
};

    One-based indexing approach

    For this approach we change the requirements and definition for T[]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>T</mi><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$T[]$ and g()<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi><mo stretchy="false">(</mo><mo stretchy="false">)</mo></math>$g()$ a little bit. This changes the implementation a little bit, and allows for a similar nice definition for g(i)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>g</mi><mo stretchy="false">(</mo><mi>i</mi><mo stretchy="false">)</mo></math>$g(i)$:

    def sum(int r):
    res = 0
    while (r > 0):
        res += t[r]
        r = g(r)

def increase(int i, int delta):
    for all j with g(j) < i <= j:

    g(7)=g(1112)=1102=6g(6)=g(1102)=1002=4g(4)=g(1002)=0002=0<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mtable displaystyle="true" columnalign="right left" columnspacing="0em" rowspacing="3pt"><mtr><mtd><mi>g</mi><mo stretchy="false">(</mo><mn>7</mn><mo stretchy="false">)</mo><mo>=</mo><mi>g</mi><mo stretchy="false">(</mo><msub><mn>111</mn><mn>2</mn></msub><mo stretchy="false">)</mo><mo>=</mo><msub><mn>110</mn><mn>2</mn></msub></mtd><mtd><mi></mi><mo>=</mo><mn>6</mn></mtd></mtr><mtr><mtd></mtd></mtr><mtr><mtd><mi>g</mi><mo stretchy="false">(</mo><mn>6</mn><mo stretchy="false">)</mo><mo>=</mo><mi>g</mi><mo stretchy="false">(</mo><msub><mn>110</mn><mn>2</mn></msub><mo stretchy="false">)</mo><mo>=</mo><msub><mn>100</mn><mn>2</mn></msub></mtd><mtd><mi></mi><mo>=</mo><mn>4</mn></mtd></mtr><mtr><mtd></mtd></mtr><mtr><mtd><mi>g</mi><mo stretchy="false">(</mo><mn>4</mn><mo stretchy="false">)</mo><mo>=</mo><mi>g</mi><mo stretchy="false">(</mo><msub><mn>100</mn><mn>2</mn></msub><mo stretchy="false">)</mo><mo>=</mo><msub><mn>000</mn><mn>2</mn></msub></mtd><mtd><mi></mi><mo>=</mo><mn>0</mn></mtd></mtr><mtr><mtd></mtd></mtr></mtable></math>$$\begin{align} g(7) = g(111_2) = 110_2 &= 6 \\\\ g(6) = g(110_2) = 100_2 &= 4 \\\\ g(4) = g(100_2) = 000_2 &= 0 \\\\ \end{align}$$

    The last set bit can be extracted using i & (i)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mtext>&nbsp;</mtext><mi mathvariant="normal">&amp;</mi><mtext>&nbsp;</mtext><mo stretchy="false">(</mo><mo></mo><mi>i</mi><mo stretchy="false">)</mo></math>$i ~\&~ (-i)$, so the operation can be expressed as:

    And it's not hard to see, that you need to change all values T[j]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>T</mi><mo stretchy="false">[</mo><mi>j</mi><mo stretchy="false">]</mo></math>$T[j]$ in the sequence i, h(i), h(h(i)), <math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo>,</mo><mtext>&nbsp;</mtext><mi>h</mi><mo stretchy="false">(</mo><mi>i</mi><mo stretchy="false">)</mo><mo>,</mo><mtext>&nbsp;</mtext><mi>h</mi><mo stretchy="false">(</mo><mi>h</mi><mo stretchy="false">(</mo><mi>i</mi><mo stretchy="false">)</mo><mo stretchy="false">)</mo><mo>,</mo><mtext>&nbsp;</mtext><mo></mo></math>$i,~ h(i),~ h(h(i)),~ \dots$ when you want to update A[j]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>A</mi><mo stretchy="false">[</mo><mi>j</mi><mo stretchy="false">]</mo></math>$A[j]$, where h(i)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>h</mi><mo stretchy="false">(</mo><mi>i</mi><mo stretchy="false">)</mo></math>$h(i)$ is defined as:

    As you can see, the main benefit of this approach is that the binary operations complement each other very nicely.

    The following implementation can be used like the other implementations, however it uses one-based indexing internally.

    struct FenwickTreeOneBasedIndexing {
    vector<int> bit;  // binary indexed tree
    int n;

    FenwickTreeOneBasedIndexing(int n) {
        this->n = n + 1;
        bit.assign(n + 1, 0);
    }

    FenwickTreeOneBasedIndexing(vector<int> a)
        : FenwickTreeOneBasedIndexing(a.size()) {
        for (size_t i = 0; i < a.size(); i++)
            add(i, a[i]);
    }

    int sum(int idx) {
        int ret = 0;
        for (++idx; idx > 0; idx -= idx & -idx)
            ret += bit[idx];
        return ret;
    }

    int sum(int l, int r) {
        return sum(r) - sum(l - 1);
    }

    void add(int idx, int delta) {
        for (++idx; idx < n; idx += idx & -idx)
            bit[idx] += delta;
    }
};

    Range operations

    A Fenwick tree can support the following range operations:

    1. Point Update and Range Query
    2. Range Update and Point Query
    3. Range Update and Range Query

    1. Point Update and Range Query

    This is just the ordinary Fenwick tree as explained above.

    2. Range Update and Point Query

    Using simple tricks we can also do the reverse operations: increasing ranges and querying for single values.

    Let the Fenwick tree be initialized with zeros. Suppose that we want to increment the interval [l,r]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mi>l</mi><mo>,</mo><mi>r</mi><mo stretchy="false">]</mo></math>$[l, r]$ by x<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>x</mi></math>$x$. We make two point update operations on Fenwick tree which are add(l, x) and add(r+1, -x).

    If we want to get the value of A[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>A</mi><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$A[i]$, we just need to take the prefix sum using the ordinary range sum method. To see why this is true, we can just focus on the previous increment operation again. If i<l<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo>&lt;</mo><mi>l</mi></math>$i < l$, then the two update operations have no effect on the query and we get the sum 0<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>0</mn></math>$0$. If i[l,r]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo></mo><mo stretchy="false">[</mo><mi>l</mi><mo>,</mo><mi>r</mi><mo stretchy="false">]</mo></math>$i \in [l, r]$, then we get the answer x<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>x</mi></math>$x$ because of the first update operation. And if i>r<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo>&gt;</mo><mi>r</mi></math>$i > r$, then the second update operation will cancel the effect of first one.

    The following implementation uses one-based indexing.

    void add(int idx, int val) {
    for (++idx; idx < n; idx += idx & -idx)
        bit[idx] += val;
}

void range_add(int l, int r, int val) {
    add(l, val);
    add(r + 1, -val);
}

int point_query(int idx) {
    int ret = 0;
    for (++idx; idx > 0; idx -= idx & -idx)
        ret += bit[idx];
    return ret;
}

    Note: of course it is also possible to increase a single point A[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>A</mi><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$A[i]$ with range_add(i, i, val).

    3. Range Update and Range Query

    To support both range updates and range queries we will use two BITs namely B1[]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>B</mi><mn>1</mn></msub><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$B_1[]$ and B2[]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>B</mi><mn>2</mn></msub><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$B_2[]$, initialized with zeros.

    Suppose that we want to increment the interval [l,r]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mi>l</mi><mo>,</mo><mi>r</mi><mo stretchy="false">]</mo></math>$[l, r]$ by the value x<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>x</mi></math>$x$. Similarly as in the previous method, we perform two point updates on B1<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>B</mi><mn>1</mn></msub></math>$B_1$: add(B1, l, x) and add(B1, r+1, -x). And we also update B2<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>B</mi><mn>2</mn></msub></math>$B_2$. The details will be explained later.

    def range_add(l, r, x):
    add(B1, l, x)
    add(B1, r+1, -x)
    add(B2, l, x*(l-1))
    add(B2, r+1, -x*r))
    After the range update (l,r,x)<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">(</mo><mi>l</mi><mo>,</mo><mi>r</mi><mo>,</mo><mi>x</mi><mo stretchy="false">)</mo></math>$(l, r, x)$ the range sum query should return the following values:

    We can write the range sum as difference of two terms, where we use B1<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>B</mi><mn>1</mn></msub></math>$B_1$ for first term and B2<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>B</mi><mn>2</mn></msub></math>$B_2$ for second term. The difference of the queries will give us prefix sum over [0,i]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mn>0</mn><mo>,</mo><mi>i</mi><mo stretchy="false">]</mo></math>$[0, i]$.

    The last expression is exactly equal to the required terms. Thus we can use B2<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>B</mi><mn>2</mn></msub></math>$B_2$ for shaving off extra terms when we multiply B1[i]i<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>B</mi><mn>1</mn></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo><mo></mo><mi>i</mi></math>$B_1[i]\times i$.

    We can find arbitrary range sums by computing the prefix sums for l1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>l</mi><mo></mo><mn>1</mn></math>$l-1$ and r<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>r</mi></math>$r$ and taking the difference of them again.

    def add(b, idx, x):
    while idx <= N:
        b[idx] += x
        idx += idx & -idx

def range_add(l,r,x):
    add(B1, l, x)
    add(B1, r+1, -x)
    add(B2, l, x*(l-1))
    add(B2, r+1, -x*r)

def sum(b, idx):
    total = 0
    while idx > 0:
        total += b[idx]
        idx -= idx & -idx
    return total

def prefix_sum(idx):
    return sum(B1, idx)*idx -  sum(B2, idx)

def range_sum(l, r):
    return prefix_sum(r) - prefix_sum(l-1)
