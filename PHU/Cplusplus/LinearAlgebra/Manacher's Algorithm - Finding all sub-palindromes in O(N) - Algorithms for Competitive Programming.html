    Manacher's Algorithm - Finding all sub-palindromes in O(N)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>N</mi><mo stretchy="false">)</mo></math>$O(N)$

    Statement

    Given string s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$ with length n<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>n</mi></math>$n$. Find all the pairs (i,j)<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">(</mo><mi>i</mi><mo>,</mo><mi>j</mi><mo stretchy="false">)</mo></math>$(i, j)$ such that substring s[ij]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi><mo stretchy="false">[</mo><mi>i</mi><mo></mo><mi>j</mi><mo stretchy="false">]</mo></math>$s[i\dots j]$ is a palindrome. String t<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>t</mi></math>$t$ is a palindrome when t=trev<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>t</mi><mo>=</mo><msub><mi>t</mi><mrow data-mjx-texclass="ORD"><mi>r</mi><mi>e</mi><mi>v</mi></mrow></msub></math>$t = t_{rev}$ (trev<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>t</mi><mrow data-mjx-texclass="ORD"><mi>r</mi><mi>e</mi><mi>v</mi></mrow></msub></math>$t_{rev}$ is a reversed string for t<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>t</mi></math>$t$).

    More precise statement

    In the worst case string might have up to O(n2)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><msup><mi>n</mi><mn>2</mn></msup><mo stretchy="false">)</mo></math>$O(n^2)$ palindromic substrings, and at the first glance it seems that there is no linear algorithm for this problem.

    But the information about the palindromes can be kept in a compact way: for each position i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ we will find the number of non-empty palindromes centered at this position.

    Palindromes with a common center form a contiguous chain, that is if we have a palindrome of length l<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>l</mi></math>$l$ centered in i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$, we also have palindromes of lengths l2<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>l</mi><mo></mo><mn>2</mn></math>$l-2$, l4<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>l</mi><mo></mo><mn>4</mn></math>$l-4$ and so on also centered in i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$. Therefore, we will collect the information about all palindromic substrings in this way.

    Palindromes of odd and even lengths are accounted for separately as dodd[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$d_{odd}[i]$ and deven[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>e</mi><mi>v</mi><mi>e</mi><mi>n</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$d_{even}[i]$. For the palindromes of even length we assume that they're centered in the position i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ if their two central characters are s[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$s[i]$ and s[i1]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi><mo stretchy="false">[</mo><mi>i</mi><mo></mo><mn>1</mn><mo stretchy="false">]</mo></math>$s[i-1]$.

    For instance, string s=abababc<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi><mo>=</mo><mi>a</mi><mi>b</mi><mi>a</mi><mi>b</mi><mi>a</mi><mi>b</mi><mi>c</mi></math>$s = abababc$ has three palindromes with odd length with centers in the position s[3]=b<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi><mo stretchy="false">[</mo><mn>3</mn><mo stretchy="false">]</mo><mo>=</mo><mi>b</mi></math>$s[3] = b$, i. e. dodd[3]=3<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mn>3</mn><mo stretchy="false">]</mo><mo>=</mo><mn>3</mn></math>$d_{odd}[3] = 3$:

    And string s=cbaabd<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi><mo>=</mo><mi>c</mi><mi>b</mi><mi>a</mi><mi>a</mi><mi>b</mi><mi>d</mi></math>$s = cbaabd$ has two palindromes with even length with centers in the position s[3]=a<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi><mo stretchy="false">[</mo><mn>3</mn><mo stretchy="false">]</mo><mo>=</mo><mi>a</mi></math>$s[3] = a$, i. e. deven[3]=2<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>e</mi><mi>v</mi><mi>e</mi><mi>n</mi></mrow></msub><mo stretchy="false">[</mo><mn>3</mn><mo stretchy="false">]</mo><mo>=</mo><mn>2</mn></math>$d_{even}[3] = 2$:

    c b a as3 bdeven[3]=2d<math xmlns="http://www.w3.org/1998/Math/MathML" display="block"><mi>c</mi><mtext>&nbsp;</mtext><mover><mrow data-mjx-texclass="OP"><mover><mrow><mi>b</mi><mtext>&nbsp;</mtext><mi>a</mi><mtext>&nbsp;</mtext><munder><mrow data-mjx-texclass="OP"><munder><mi>a</mi><mo></mo></munder></mrow><mrow data-mjx-texclass="ORD"><msub><mi>s</mi><mn>3</mn></msub></mrow></munder><mtext>&nbsp;</mtext><mi>b</mi></mrow><mo></mo></mover></mrow><mrow data-mjx-texclass="ORD"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>e</mi><mi>v</mi><mi>e</mi><mi>n</mi></mrow></msub><mo stretchy="false">[</mo><mn>3</mn><mo stretchy="false">]</mo><mo>=</mo><mn>2</mn></mrow></mover><mi>d</mi></math>$$c\ \overbrace{b\ a\ \underbrace{a}_{s_3}\ b}^{d_{even}[3]=2} d$$

    It's a surprising fact that there is an algorithm, which is simple enough, that calculates these "palindromity arrays" dodd[]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$d_{odd}[]$ and deven[]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>e</mi><mi>v</mi><mi>e</mi><mi>n</mi></mrow></msub><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$d_{even}[]$ in linear time. The algorithm is described in this article.

    Solution

    In general, this problem has many solutions: with String Hashing it can be solved in O(nlogn)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mo></mo><mi>log</mi><mo data-mjx-texclass="NONE"></mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n\cdot \log n)$, and with Suffix Trees and fast LCA this problem can be solved in O(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n)$.

    But the method described here is sufficiently simpler and has less hidden constant in time and memory complexity. This algorithm was discovered by Glenn K. Manacher in 1975.

    Another modern way to solve this problem and to deal with palindromes in general is through the so-called palindromic tree, or eertree.

    Trivial algorithm

    To avoid ambiguities in the further description we denote what "trivial algorithm" is.

    It's the algorithm that does the following. For each center position i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ it tries to increase the answer by one as long as it's possible, comparing a pair of corresponding characters each time.

    Such an algorithm is slow, it can calculate the answer only in O(n2)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><msup><mi>n</mi><mn>2</mn></msup><mo stretchy="false">)</mo></math>$O(n^2)$.

    The implementation of the trivial algorithm is:

    vector<int> manacher_odd(string s) {
    int n = s.size();
    s = "$" + s + "^";
    vector<int> p(n + 2);
    for(int i = 1; i <= n; i++) {
        while(s[i - p[i]] == s[i + p[i]]) {
            p[i]++;
        }
    }
    return vector<int>(begin(p) + 1, end(p) - 1);
}

    Terminal characters $ and ^ were used to avoid dealing with ends of the string separately.

    Manacher's algorithm

    We describe the algorithm to find all the sub-palindromes with odd length, i. e. to calculate dodd[]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$d_{odd}[]$.

    For fast calculation we'll maintain the borders (l,r)<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">(</mo><mi>l</mi><mo>,</mo><mi>r</mi><mo stretchy="false">)</mo></math>$(l, r)$ of the rightmost found (sub-)palindrome (i. e. the current rightmost (sub-)palindrome is s[l+1]s[l+2]s[r1]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi><mo stretchy="false">[</mo><mi>l</mi><mo>+</mo><mn>1</mn><mo stretchy="false">]</mo><mi>s</mi><mo stretchy="false">[</mo><mi>l</mi><mo>+</mo><mn>2</mn><mo stretchy="false">]</mo><mo></mo><mi>s</mi><mo stretchy="false">[</mo><mi>r</mi><mo></mo><mn>1</mn><mo stretchy="false">]</mo></math>$s[l+1] s[l+2] \dots s[r-1]$). Initially we set l=0,r=1<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>l</mi><mo>=</mo><mn>0</mn><mo>,</mo><mi>r</mi><mo>=</mo><mn>1</mn></math>$l = 0, r = 1$, which corresponds to the empty string.

    So, we want to calculate dodd[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$d_{odd}[i]$ for the next i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$, and all the previous values in dodd[]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$d_{odd}[]$ have been already calculated. We do the following:

    • If i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ is outside the current sub-palindrome, i. e. ir<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo></mo><mi>r</mi></math>$i \geq r$, we'll just launch the trivial algorithm.

      So we'll increase dodd[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$d_{odd}[i]$ consecutively and check each time if the current rightmost substring [idodd[i]i+dodd[i]]<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">[</mo><mi>i</mi><mo></mo><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo><mo></mo><mi>i</mi><mo>+</mo><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo><mo stretchy="false">]</mo></math>$[i - d_{odd}[i]\dots i + d_{odd}[i]]$ is a palindrome. When we find the first mismatch or meet the boundaries of s<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>s</mi></math>$s$, we'll stop. In this case we've finally calculated dodd[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$d_{odd}[i]$. After this, we must not forget to update (l,r)<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">(</mo><mi>l</mi><mo>,</mo><mi>r</mi><mo stretchy="false">)</mo></math>$(l, r)$. r<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>r</mi></math>$r$ should be updated in such a way that it represents the last index of the current rightmost sub-palindrome.

    • Now consider the case when ir<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo></mo><mi>r</mi></math>$i \le r$. We'll try to extract some information from the already calculated values in dodd[]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$d_{odd}[]$. So, let's find the "mirror" position of i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ in the sub-palindrome (l,r)<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">(</mo><mi>l</mi><mo>,</mo><mi>r</mi><mo stretchy="false">)</mo></math>$(l, r)$, i.e. we'll get the position j=l+(ri)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>j</mi><mo>=</mo><mi>l</mi><mo>+</mo><mo stretchy="false">(</mo><mi>r</mi><mo></mo><mi>i</mi><mo stretchy="false">)</mo></math>$j = l + (r - i)$, and we check the value of dodd[j]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>j</mi><mo stretchy="false">]</mo></math>$d_{odd}[j]$. Because j<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>j</mi></math>$j$ is the position symmetrical to i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ with respect to (l+r)/2<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">(</mo><mi>l</mi><mo>+</mo><mi>r</mi><mo stretchy="false">)</mo><mrow data-mjx-texclass="ORD"><mo>/</mo></mrow><mn>2</mn></math>$(l+r)/2$, we can almost always assign dodd[i]=dodd[j]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo><mo>=</mo><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>j</mi><mo stretchy="false">]</mo></math>$d_{odd}[i] = d_{odd}[j]$.       But there is a tricky case to be handled correctly: when the "inner" palindrome reaches the borders of the "outer" one, i. e. jdodd[j]l<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>j</mi><mo></mo><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>j</mi><mo stretchy="false">]</mo><mo></mo><mi>l</mi></math>$j - d_{odd}[j] \le l$ (or, which is the same, i+dodd[j]r<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi><mo>+</mo><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>j</mi><mo stretchy="false">]</mo><mo></mo><mi>r</mi></math>$i + d_{odd}[j] \ge r$). Because the symmetry outside the "outer" palindrome is not guaranteed, just assigning dodd[i]=dodd[j]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo><mo>=</mo><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>j</mi><mo stretchy="false">]</mo></math>$d_{odd}[i] = d_{odd}[j]$ will be incorrect: we do not have enough data to state that the palindrome in the position i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ has the same length.

After this we'll run the trivial algorithm which will try to increase dodd[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$d_{odd}[i]$ while it's possible.

    Complexity of Manacher's algorithm

    At the first glance it's not obvious that this algorithm has linear time complexity, because we often run the naive algorithm while searching the answer for a particular position.

    However, a more careful analysis shows that the algorithm is linear. In fact, Z-function building algorithm, which looks similar to this algorithm, also works in linear time.

    We can notice that every iteration of trivial algorithm increases r<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>r</mi></math>$r$ by one. Also r<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>r</mi></math>$r$ cannot be decreased during the algorithm. So, trivial algorithm will make O(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n)$ iterations in total.

    Other parts of Manacher's algorithm work obviously in linear time. Thus, we get O(n)<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>O</mi><mo stretchy="false">(</mo><mi>n</mi><mo stretchy="false">)</mo></math>$O(n)$ time complexity.

    Implementation of Manacher's algorithm

    For calculating dodd[]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mo stretchy="false">]</mo></math>$d_{odd}[]$, we get the following code. Things to note:

    • i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ is the index of the center letter of the current palindrome.
    • If i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ exceeds r<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>r</mi></math>$r$, dodd[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$d_{odd}[i]$ is initialized to 0.
    • If i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ does not exceed r<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>r</mi></math>$r$, dodd[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$d_{odd}[i]$ is either initialized to the dodd[j]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>j</mi><mo stretchy="false">]</mo></math>$d_{odd}[j]$, where j<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>j</mi></math>$j$ is the mirror position of i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ in (l,r)<math xmlns="http://www.w3.org/1998/Math/MathML"><mo stretchy="false">(</mo><mi>l</mi><mo>,</mo><mi>r</mi><mo stretchy="false">)</mo></math>$(l,r)$, or dodd[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$d_{odd}[i]$ is restricted to the size of the "outer" palindrome.
    • The while loop denotes the trivial algorithm. We launch it irrespective of the value of k<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>k</mi></math>$k$.
    • If the size of palindrome centered at i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ is x<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>x</mi></math>$x$, then dodd[i]<math xmlns="http://www.w3.org/1998/Math/MathML"><msub><mi>d</mi><mrow data-mjx-texclass="ORD"><mi>o</mi><mi>d</mi><mi>d</mi></mrow></msub><mo stretchy="false">[</mo><mi>i</mi><mo stretchy="false">]</mo></math>$d_{odd}[i]$ stores x+12<math xmlns="http://www.w3.org/1998/Math/MathML"><mfrac><mrow><mi>x</mi><mo>+</mo><mn>1</mn></mrow><mn>2</mn></mfrac></math>$\frac{x+1}{2}$.
    vector<int> manacher_odd(string s) {
    int n = s.size();
    s = "$" + s + "^";
    vector<int> p(n + 2);
    int l = 1, r = 1;
    for(int i = 1; i <= n; i++) {
        p[i] = max(0, min(r - i, p[l + (r - i)]));
        while(s[i - p[i]] == s[i + p[i]]) {
            p[i]++;
        }
        if(i + p[i] > r) {
            l = i - p[i], r = i + p[i];
        }
    }
    return vector<int>(begin(p) + 1, end(p) - 1);
}

    Working with parities

    Although it is possible to implement Manacher's algorithm for odd and even lengths separately, the implementation of the version for even lengths is often deemed more difficult, as it is less natural and easily leads to off-by-one errors.

    To mitigate this, it is possible to reduce the whole problem to the case when we only deal with the palindromes of odd length. To do this, we can put an additional # character between each letter in the string and also in the beginning and the end of the string:

    Note that d[2i]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>d</mi><mo stretchy="false">[</mo><mn>2</mn><mi>i</mi><mo stretchy="false">]</mo></math>$d[2i]$ and d[2i+1]<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>d</mi><mo stretchy="false">[</mo><mn>2</mn><mi>i</mi><mo>+</mo><mn>1</mn><mo stretchy="false">]</mo></math>$d[2i+1]$ are essentially the increased by 1<math xmlns="http://www.w3.org/1998/Math/MathML"><mn>1</mn></math>$1$ lengths of the largest odd- and even-length palindromes centered in i<math xmlns="http://www.w3.org/1998/Math/MathML"><mi>i</mi></math>$i$ correspondingly.

    The reduction is implemented in the following way:

    vector<int> manacher(string s) {
    string t;
    for(auto c: s) {
        t += string("#") + c;
    }
    auto res = manacher_odd(t + "#");
    return vector<int>(begin(res) + 1, end(res) - 1);
}
